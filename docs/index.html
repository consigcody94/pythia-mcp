01

What you are looking at

The Higgs boson couples to every massive particle. By measuring how often it is produced and how it decays, we test whether those couplings match the Standard Model — and where they don't, new physics may hide.

The observable

Signal strength μ

μ = σobs / σSM

The ratio of an observed rate to the Standard-Model prediction. μ = 1 means "exactly as predicted." Every point on the forest plot below is one such measurement, with its asymmetric uncertainty.

The framework

κ — coupling modifiers

κi = gi / giSM

Each coupling is scaled by a factor κ. Rates go as κ², so a 10% coupling shift is a ~20% rate shift. Set every κ = 1 and you recover the Standard Model.

The engine

Lilith likelihood

−2 ln L(κ)

Lilith compares a coupling hypothesis against this whole database and returns a likelihood. Pythia exposes that to AI assistants over the Model Context Protocol.

03

The coupling laboratory

Move the two master couplings — κV for the force carriers (W, Z) and κF for the fermions (quarks, leptons) — and watch the predicted signal strengths shift. This is the leading-order κ-framework; the full fit is what Lilith does.

Derived: H→γγ runs on the interference κγ ≈ |1.26 κV − 0.26 κF|; the total width rescales by Σ BRi κi². At κV=κF=1 every channel returns μ = 1.

Pythia — the MCP layer

Pythia wraps Lilith as a Model Context Protocol server, so an AI assistant can run real Higgs phenomenology in conversation. Recently it grew a genuine data-ingestion pipeline.

Field note · why ingestion is hard

HEPData's search and table-download endpoints sit behind a Cloudflare bot challenge, so they can't be reached from an automated server. Pythia handles this honestly: discovery falls back to INSPIRE-HEP, and the converter accepts a downloaded table JSON so it works fully offline — turning a fetched measurement into a Lilith <expmu> file the engine can actually use.
